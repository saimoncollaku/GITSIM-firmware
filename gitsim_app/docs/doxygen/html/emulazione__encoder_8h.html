void aggiorna_passo_encoder1 ( void )

Aggiorna il passo dell'encoder e_1.

Questa funzione calcola e aggiorna la lunghezza del passo dell'encoder e_1 basandosi sul suo diametro e sul numero di impulsi per rivoluzione (ppr).

La formula utilizzata è:

\[ l_{passo} = \frac{diametro \cdot \pi}{ppr \cdot 2} \]

dove:

  • \(l_{passo}\) è il passo dell'encoder
  • \(diametro\) è il diametro della ruota dell'encoder
  • \(\pi\) è il valore di PI greco
  • \(ppr\) è il numero di impulsi per rivoluzione dell'encoder
  • Il fattore 2 nel denominatore è usato per ottenere la risoluzione x2 del passo

◆ emula_sensori_encoder()

void emula_sensori_encoder ( void )

Emula le uscite degli encoder e_1 ed e_2.

Questa funzione gestisce l'emulazione dei sensori per gli encoder e_1 ed e_2. L'emulazione viene eseguita solo se l'applicazione GITSIM è connessa, altrimenti la funzione non esegue alcuna operazione.

Nota
L'emulazione dei sensori è utile per test e debug, permettendo di simulare il comportamento degli encoder senza hardware fisico.

◆ inizializza_variabili_encoder()

void inizializza_variabili_encoder ( void )

Inizializza le variabili e i GPIO per gli encoder.

Questa funzione esegue le seguenti operazioni:

  1. Imposta il tempo di aggiornamento (t_update)
  2. Inizializza le strutture degli encoder e_1 e e_2
  3. Inizializza i GPIO associati a ciascun encoder
  4. Resetta lo stato dei GPIO
Nota
  • Utilizza funzioni esterne come ritorna_tempo_del_polling()
  • Utilizza costanti XPAR per l'identificazione dei dispositivi GPIO
Si veda anche
inizializza_encoder, XGpio_Initialize, reset_gpio
